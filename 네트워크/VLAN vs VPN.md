## VLAN vs VPN

### 1. VLAN (Virtual LAN)

#### 정의

VLAN은 물리적으로 하나의 네트워크를 **논리적으로 여러 개의 네트워크로 분리**하는 기술이다.
같은 스위치 내에서 포트를 그룹화하여, 서로 다른 VLAN 간에는 브로드캐스트가 전달되지 않도록 한다.

#### 주요 특징

* OSI **2계층 (데이터 링크 계층)** 에서 동작
* **VLAN ID (태그)** 를 이용해 트래픽을 구분
* **브로드캐스트 도메인 분리**로 네트워크 효율성 향상
* 스위치 기반 네트워크에서 주로 사용
* 암호화 기능은 제공하지 않음

#### 사용 예시

* 부서별 네트워크 분리 (예: 개발팀 VLAN10, 영업팀 VLAN20)
* 내부망 보안 및 트래픽 관리

---

### 2. VPN (Virtual Private Network)

#### 정의

VPN은 인터넷과 같은 공용망을 통해 **사설망처럼 안전하게 통신할 수 있도록 하는 기술**이다.
데이터를 암호화하고, 터널링을 통해 외부에서도 내부 네트워크에 접근할 수 있게 한다.

#### 주요 특징

* OSI **3계층 (네트워크 계층)** 에서 동작
* **암호화 및 터널링** 기술을 이용해 안전한 통신 제공
* 원격 근무나 외부 접속 시 내부망과 같은 환경 제공
* VPN 서버 및 클라이언트 필요

#### 사용 예시

* 재택근무 시 회사 내부망 접속
* 공용 Wi-Fi 환경에서 안전한 통신 확보

---

### 3. VLAN vs VPN 비교

| 구분        | VLAN (Virtual LAN)     | VPN (Virtual Private Network)   |
| --------- | ---------------------- | ------------------------------- |
| **기본 개념** | 하나의 물리적 네트워크를 논리적으로 분리 | 공용망을 이용해 사설망처럼 통신               |
| **목적**    | 내부망 분리 및 트래픽 관리        | 외부에서 안전하게 내부망 접근                |
| **동작 계층** | 2계층 (데이터 링크 계층)        | 3계층 (네트워크 계층)                   |
| **보안성**   | 논리적 분리 수준, 암호화 없음      | 암호화된 통신으로 높은 보안성                |
| **사용 기술** | VLAN ID(Tag), 스위칭      | 터널링(Tunneling), 암호화(Encryption) |
| **대표 장비** | 스위치, 라우터               | VPN 서버, 클라이언트                   |
| **통신 방식** | 같은 스위치 내 논리적 구획        | 인터넷을 통한 암호화된 연결                 |
| **사용 목적** | 내부 트래픽 효율화, 부서 간 격리    | 외부 접속 보호, 안전한 데이터 전송            |

---


* **VLAN**은 내부 네트워크를 **논리적으로 분리**하는 기술이다.
* **VPN**은 외부 사용자가 **공용망을 통해 내부망에 안전하게 접근**할 수 있도록 하는 기술이다.
* VLAN은 내부 트래픽 관리 중심, VPN은 보안 통신 중심이다.
