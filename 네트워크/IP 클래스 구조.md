# IP 클래스 구조 (A, B, C 클래스)

IPv4 주소는 32비트(4옥텟)로 구성되어 있으며,
이 중 일부는 네트워크를 구분하기 위한 **네트워크 부분**,     
나머지는 네트워크 내의 개별 기기를 구분하는 **호스트 부분**으로 사용된다.

IP 클래스는 주소 공간을 효율적으로 나누기 위해 A, B, C 등으로 구분된다.

---

## 1. 클래스별 기본 구분

| 클래스 | 2진수 시작 패턴 | 첫 옥텟 범위(10진수) | 네트워크 규모 |
| :-: | :-------: | :-----------: | :-----: |
|  A  |  0xxxxxxx |    0 ~ 127    | 대형 네트워크 |
|  B  |  10xxxxxx |   128 ~ 191   | 중형 네트워크 |
|  C  |  110xxxxx |   192 ~ 223   | 소형 네트워크 |

---

## 2. 클래스별 비트 구성

| 클래스 | 시작 비트 |  네트워크 비트 수  | 호스트 비트 수 |     네트워크 개수     |         호스트 수        |
| :-: | :---: | :---------: | :------: | :-------------: | :------------------: |
|  A  |   0   |  8 (1 + 7)  |    24    |     2⁷ = 128    | 2²⁴ - 2 ≈ 16,777,214 |
|  B  |   10  | 16 (2 + 14) |    16    |   2¹⁴ = 16,384  |   2¹⁶ - 2 = 65,534   |
|  C  |  110  | 24 (3 + 21) |     8    | 2²¹ ≈ 2,097,152 |     2⁸ - 2 = 254     |

※ `-2`는 네트워크 주소(모두 0)와 브로드캐스트 주소(모두 1)를 제외하기 때문이다.

---

## 3. 클래스별 특징

### A 클래스

* 첫 비트가 `0`으로 시작.
* 대형 조직이나 ISP가 사용.
* 네트워크 수는 적지만, 하나의 네트워크당 호스트 수가 매우 많다.
* 범위: `0.0.0.0 ~ 127.255.255.255`

### B 클래스

* 시작 비트는 `10`.
* 중간 규모의 기관이나 기업에서 사용.
* 네트워크 수와 호스트 수의 균형이 적절하다.
* 범위: `128.0.0.0 ~ 191.255.255.255`

### C 클래스

* 시작 비트는 `110`.
* 소규모 네트워크(가정, 소기업)에서 주로 사용.
* 네트워크 수는 많지만, 네트워크당 호스트 수는 적다.
* 범위: `192.0.0.0 ~ 223.255.255.255`

---

## 4. 사설 IP와의 관계

사설 IP는 인터넷 외부와 직접 연결되지 않는 내부망 전용 주소로,
각 클래스별로 일부 범위를 내부용으로 예약해 두었다.

| 클래스 |            사설 IP 범위           |      비고     |
| :-: | :---------------------------: | :---------: |
|  A  |   10.0.0.0 ~ 10.255.255.255   |   대규모 내부망   |
|  B  |  172.16.0.0 ~ 172.31.255.255  |    중형 내부망   |
|  C  | 192.168.0.0 ~ 192.168.255.255 | 가정·소규모 네트워크 |

---

## 5. 핵심 요약

* IP 주소는 32비트이며, 앞부분은 네트워크 식별, 뒷부분은 호스트 식별에 사용된다.
* 클래스 구분은 IP의 앞 비트(2진수 패턴)에 따라 정해진다.
* 각 클래스는 네트워크 수와 호스트 수의 비율이 다르며,
  C 클래스는 가장 소규모 네트워크에 적합하다.
