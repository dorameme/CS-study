## 접근 통제 모델 정리

### 1. DAC (Discretionary Access Control, 임의적 접근 통제)

#### 개념

DAC는 **자원의 소유자(Owner)** 가 접근 권한을 임의로 부여하거나 회수할 수 있는 방식이다.     
즉, 파일이나 디렉터리의 접근 권한을 사용자가 직접 설정할 수 있다.

#### 특징

* 사용자가 권한을 **자율적으로 설정 가능**
* 윈도우, 유닉스의 파일 권한(ACL)이 대표적인 예
* 관리가 간편하지만, **권한 누적 및 오남용 위험**이 있음

#### 예시

```text
파일 소유자가 다른 사용자에게 읽기(read) 권한을 직접 부여하거나 제거함
```

---

### 2. MAC (Mandatory Access Control, 강제적 접근 통제)

#### 개념

MAC은 **시스템이 중앙에서 접근 정책을 통제**하며,      
사용자는 자신의 권한을 임의로 변경할 수 없는 방식이다.      
주로 **군사나 정부기관의 기밀 정보 보호**에 사용된다.

#### 특징

* 보안 등급(Label)을 기준으로 접근 결정
* 사용자와 자원 모두에 보안 등급이 부여됨
* 정책은 **시스템 관리자나 보안 정책에 의해 고정**됨
* 높은 보안성을 제공하지만 유연성이 낮음

#### 예시

```text
"기밀(Secret)" 등급 사용자는 "일반(Unclassified)" 문서를 읽을 수 있지만,
"일반" 사용자는 "기밀" 문서를 열람할 수 없음
```

---

### 3. RBAC (Role-Based Access Control, 역할 기반 접근 통제)

#### 개념

RBAC는 사용자에게 직접 권한을 부여하지 않고,        
**역할(Role)** 에 따라 권한을 관리하는 방식이다.     
사용자는 역할을 부여받고, 역할에 해당하는 권한을 행사한다.

#### 특징

* 권한 부여가 역할 중심으로 이뤄져 **관리 효율성**이 높음
* 기업, 조직 등에서 직무 기반 권한 관리에 적합
* 역할 변경만으로 손쉽게 권한 조정 가능

#### 예시

```text
관리자(Admin): 모든 데이터 접근 가능  
일반 사용자(User): 자신의 데이터만 접근 가능
```

### 4. Bell-LaPadula 모델 (기밀성 중심)

#### 개념

Bell-LaPadula 모델은 **기밀성(Confidentiality)** 을 보호하기 위한 **MAC 기반 보안 모델**이다.     
주로 군사 보안 시스템에서 사용된다.

#### 주요 규칙

* **No Read Up (상위 등급 읽기 금지)**
  낮은 등급 사용자는 높은 등급의 정보를 읽을 수 없다.
* **No Write Down (하위 등급 쓰기 금지)**
  높은 등급 사용자는 낮은 등급의 영역에 정보를 쓸 수 없다.
* 목적: **정보의 기밀성 유출 방지**

#### 예시

```text
비밀(Secret) 문서를 일반 사용자(Unclassified)가 읽을 수 없음
```

---

### 5. Biba 모델 (무결성 중심)

#### 개념

Biba 모델은 **무결성(Integrity)** 을 보호하기 위한 보안 모델이다.
Bell-LaPadula의 반대 개념으로 볼 수 있다.

#### 주요 규칙

* **No Read Down (하위 등급 읽기 금지)**
  높은 신뢰 등급 사용자는 낮은 신뢰 등급 데이터를 읽을 수 없다.
* **No Write Up (상위 등급 쓰기 금지)**
  낮은 신뢰 등급 사용자는 높은 신뢰 등급 데이터에 쓸 수 없다.
* 목적: **데이터의 신뢰성 유지 및 변조 방지**

#### 예시

```text
관리자는 신뢰할 수 없는 사용자 데이터(하위 등급)를 읽을 수 없음
```

### 6. 비교 요약

| 구분                | 모델             | 통제 주체 | 주요 개념                      | 보안 목표  | 예시                  |
| ----------------- | -------------- | ----- | -------------------------- | ------ | ------------------- |
| **DAC**           | 임의적 접근 통제      | 사용자   | 소유자가 권한 부여                 | 편의성    | 파일 소유자가 접근 권한 직접 설정 |
| **MAC**           | 강제적 접근 통제      | 시스템   | 등급 기반 접근 제어                | 기밀성    | 정부/군사 시스템           |
| **RBAC**          | 역할 기반 접근 통제    | 관리자   | 역할(Role) 기반 권한 부여          | 관리 효율성 | 기업 내부 역할 관리         |
| **Bell-LaPadula** | 보안 모델 (MAC 기반) | 시스템   | No Read Up / No Write Down | 기밀성    | 비밀문서 열람 통제          |
| **Biba**          | 보안 모델 (MAC 기반) | 시스템   | No Read Down / No Write Up | 무결성    | 데이터 변조 방지           |

---

### 요약

* **DAC**: 사용자 자율 설정 (편리하지만 보안 취약)
* **MAC**: 시스템 중심 통제 (보안 강력, 유연성 낮음)
* **RBAC**: 역할 기반 효율적 관리 (조직형 시스템에 적합)
* **Bell-LaPadula**: 기밀성 중심 MAC 모델
* **Biba**: 무결성 중심 MAC 모델
